Understanding CSS

An introduction covering what css is, why we use it, who inverted it, versions, responsive design, layouts, and styling methods.

What is CSS?

CSS stands for Cascading style sheets. it is a style sheet language used to describe the presentation(look and feel) of a document written in HTML, It controls responsiveness, etc.

symply put: HTML is the structure(skeleton),CSS is the style (clothes/skin)

Why is CSS used?

  • separation of content & design -> HTML handles content, CSS handles design.
  • Reusability -> one CSS file can style multiple HTML pages.
  • Consistency -> Keeps the look & feel uniform across a website.
  • Responsive design -> Makes website adapt to different devices(pc,Tablet,Mobile).
  • Faster development -> Change style in one place and it reflects everywhere.

Who invented CSS?

Hakon Wium Lie and Bert Bos invented CSS in 1994

Present version of CSS

The latest stable version is CSS3(introduced between 1999-2011). CSS3 is modular, divided in to parts like selectors, Backgrounds, Animations, Flexbox, and Grid. Work continues with new features (sometimes informally called CSS4), but there is no single official CSS4 version

Responsive Web Design (RWD)

Responsive design ensures websites adjust automatically to desktops, tablets, and phones. Achieved using

  • Media queries(@media screen and (max-width:768px) { .... })
  • Flexible layouts(Flexbox, CSS Grid)
  • Fluid images & percentages instead of fixed pixels

CSS Layouts

  1. Normal Flow (default)
  2. Box Model (margin, border, padding, content)
  3. Positioning (static, relative, absolute, fixed, sticky)
  4. Flexbox(1D layout - row or coiumn)
  5. CSS Grid (2D layout - rows + columns)
  6. Responsive layouts with media queries

Styling Methods in CSS

1. Inline CSS

written inside the HTML tag using the style attribute. useful for quick testing, but not recommended for large projects.

2. Intenal CSS

Defined inside a style tag within the HTML file (inside head). Good for small projects or single-page sites

3. External CSS

Written in a separate CSS file and linked with a tag. Best practice for large websites and reusability.
